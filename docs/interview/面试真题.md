## JavaScript 基础

### 1. JavaScript 的基本类型和引用类型有哪些？它们的区别是什么？

**基本类型（Primitive Types）：**
- `string` - 字符串
- `number` - 数字
- `boolean` - 布尔值
- `undefined` - 未定义
- `null` - 空值
- `symbol` - 符号（ES6 新增）
- `bigint` - 大整数（ES2020 新增）

**引用类型（Reference Types）：**
- `Object` - 对象
- `Array` - 数组
- `Function` - 函数
- `Date` - 日期
- `RegExp` - 正则表达式
- `Map` / `Set` - ES6 新增数据结构

**核心区别：**

| 特性 | 基本类型 | 引用类型 |
|------|----------|----------|
| **存储位置** | 栈内存（Stack） | 堆内存（Heap），栈中存引用地址 |
| **访问方式** | 按值访问 | 按引用访问 |
| **复制行为** | 完全拷贝，互不影响 | 拷贝引用地址，指向同一对象 |
| **比较方式** | 比较值 | 比较引用地址 |
| **可变性** | 不可变（immutable） | 可变（mutable） |

**示例代码：**

```javascript
// 基本类型：值拷贝
let a = 10;
let b = a;
b = 20;
console.log(a); // 10（不受影响）
console.log(b); // 20

// 引用类型：引用拷贝
let obj1 = { name: 'Alice' };
let obj2 = obj1;
obj2.name = 'Bob';
console.log(obj1.name); // 'Bob'（被修改了）
console.log(obj2.name); // 'Bob'

// 比较差异
console.log(1 === 1);           // true（值相等）
console.log({} === {});         // false（不同引用）
console.log([] === []);         // false（不同引用）

// 深拷贝解决方案
const deepClone = (obj) => JSON.parse(JSON.stringify(obj));
// 或使用结构化克隆
const deepClone2 = structuredClone(obj);
```

**面试要点：**
- 基本类型存储在栈中，访问速度快
- 引用类型的变量在栈中存储的是堆内存的地址指针
- 浅拷贝只拷贝第一层，深拷贝递归拷贝所有层级
- `typeof null === 'object'` 是历史遗留 bug

### 2. 已知 `a = [1,2,3]`；求 `console.log(a.push(5))` 的值是？为什么？

**答案：** 输出结果为 `4`

**解析：**
- `push()` 方法会向数组末尾添加一个或多个元素，并**返回修改后数组的新长度**
- `a.push(5)` 会将 5 添加到数组 `[1,2,3]` 中，数组变为 `[1,2,3,5]`
- 返回值为新数组的长度，即 `4`
- 注意：`push()` 方法会修改原数组

```javascript
let a = [1, 2, 3];
let result = a.push(5);
console.log(result); // 4
console.log(a); // [1, 2, 3, 5]
```

### 2. 数组排序方法除 `sort` 排序外还有哪些？并写出其中一种排序方法？

**其他排序方法：**
- 冒泡排序（Bubble Sort）
- 选择排序（Selection Sort）
- 插入排序（Insertion Sort）
- 快速排序（Quick Sort）
- 归并排序（Merge Sort）
- 堆排序（Heap Sort）
- 计数排序（Counting Sort）

**示例：快速排序实现**

```javascript
function quickSort(arr) {
  // 递归出口
  if (arr.length <= 1) {
    return arr;
  }
  
  // 选择基准值（中间位置）
  const pivotIndex = Math.floor(arr.length / 2);
  const pivot = arr.splice(pivotIndex, 1)[0];
  
  // 定义左右数组
  const left = [];
  const right = [];
  
  // 分区操作
  for (let i = 0; i < arr.length; i++) {
    if (arr[i] < pivot) {
      left.push(arr[i]);
    } else {
      right.push(arr[i]);
    }
  }
  
  // 递归排序并合并
  return quickSort(left).concat([pivot], quickSort(right));
}

// 测试
const arr = [5, 2, 8, 1, 9, 3];
console.log(quickSort(arr)); // [1, 2, 3, 5, 8, 9]
```

### 3. localStorage、sessionStorage 和 Cookie 的区别是什么？

**三者的对比：**

| 特性 | localStorage | sessionStorage | Cookie |
|------|--------------|----------------|--------|
| **生命周期** | 永久存储，除非手动删除 | 会话级，关闭标签页即清除 | 可设置过期时间，默认会话级 |
| **存储大小** | 约 5-10 MB | 约 5-10 MB | 约 4 KB |
| **与服务端通信** | 不参与 HTTP 请求 | 不参与 HTTP 请求 | 每次请求自动携带 |
| **API 易用性** | 简单易用 | 简单易用 | 较复杂，需手动解析 |
| **作用域** | 同源窗口共享 | 仅当前标签页有效 | 同源窗口共享，可设置 path/domain |
| **安全性** | 易受 XSS 攻击 | 易受 XSS 攻击 | 可设置 HttpOnly 防 XSS |

**使用示例：**

```javascript
// localStorage 用法
localStorage.setItem('username', 'Alice');
const user = localStorage.getItem('username');
localStorage.removeItem('username');
localStorage.clear(); // 清空所有

// sessionStorage 用法（API 相同）
sessionStorage.setItem('token', 'abc123');
const token = sessionStorage.getItem('token');

// Cookie 用法
document.cookie = 'userId=123; expires=Thu, 01 Jan 2027 00:00:00 GMT; path=/';

// 读取 Cookie（需要手动解析）
function getCookie(name) {
  const cookies = document.cookie.split('; ');
  for (let cookie of cookies) {
    const [key, value] = cookie.split('=');
    if (key === name) return decodeURIComponent(value);
  }
  return null;
}
```

**应用场景选择：**

- **localStorage**：
  - 用户偏好设置（主题、语言）
  - 缓存不敏感数据
  - 离线数据存储
  
- **sessionStorage**：
  - 表单临时数据（防止刷新丢失）
  - 单页应用的状态保持
  - 敏感但不需持久化的数据
  
- **Cookie**：
  - 用户认证（Session ID、JWT）
  - 需要服务端读取的数据
  - 跨子域名共享数据（设置 domain）

**安全注意事项：**

```javascript
// ❌ 不安全：存储敏感信息
localStorage.setItem('password', '123456');

// ✅ 推荐：敏感信息使用 HttpOnly Cookie
// 后端设置：Set-Cookie: session_id=xxx; HttpOnly; Secure; SameSite=Strict

// 防御 XSS 攻击
function escapeHtml(str) {
  const div = document.createElement('div');
  div.textContent = str;
  return div.innerHTML;
}

// 存储前转义
localStorage.setItem('userInput', escapeHtml(userInput));
```

**面试加分点：**
- Cookie 的 `HttpOnly` 属性可防止 JavaScript 访问，抵御 XSS
- Cookie 的 `Secure` 属性确保只在 HTTPS 下传输
- Cookie 的 `SameSite` 属性可防止 CSRF 攻击
- `storage` 事件可监听其他标签页的 localStorage 变化
- IndexedDB 适合存储大量结构化数据（>10MB）

## 事件处理与防抖节流

### 4. 常用的防止按钮多次点击触发的方案有哪些？并说明他们的区别？

**常见方案：**

#### 方案一：防抖（Debounce）
```javascript
function debounce(fn, delay = 300) {
  let timer = null;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

// 使用
button.addEventListener('click', debounce(handleClick, 300));
```

#### 方案二：节流（Throttle）
```javascript
function throttle(fn, interval = 300) {
  let lastTime = 0;
  return function(...args) {
    const now = Date.now();
    if (now - lastTime >= interval) {
      lastTime = now;
      fn.apply(this, args);
    }
  };
}

// 使用
button.addEventListener('click', throttle(handleClick, 300));
```

#### 方案三：标志位控制
```javascript
let isClicked = false;

button.addEventListener('click', function() {
  if (isClicked) return;
  isClicked = true;
  
  handleClick();
  
  // 根据业务需求重置标志位
  setTimeout(() => {
    isClicked = false;
  }, 1000);
});
```

#### 方案四：禁用按钮
```javascript
button.addEventListener('click', function() {
  button.disabled = true;
  button.textContent = '处理中...';
  
  handleClick().finally(() => {
    button.disabled = false;
    button.textContent = '提交';
  });
});
```

**区别对比：**

| 方案 | 特点 | 适用场景 |
|------|------|----------|
| **防抖** | n 秒内只执行最后一次，重新触发会重置计时器 | 搜索框输入、窗口 resize |
| **节流** | n 秒内只执行一次，时间间隔固定 | 滚动加载、按钮点击频率限制 |
| **标志位** | 简单直接，需手动控制状态重置 | 简单的防重复提交 |
| **禁用按钮** | 用户体验好，直观显示状态 | 表单提交、异步请求场景 |

## 安全与性能优化

### 5. 前端常见的攻击方式有哪些？

#### （1）XSS（跨站脚本攻击）
**原理：** 注入恶意脚本代码到网页中执行

**防御措施：**
- 对用户输入进行 HTML 实体编码
- 使用 CSP（内容安全策略）
- 设置 HttpOnly Cookie
- 使用现代框架的自动转义功能

```javascript
// XSS 攻击示例
// 用户输入：<script>alert('xss')</script>
// 应该转义为：&lt;script&gt;alert('xss')&lt;/script&gt;

function escapeHtml(str) {
  const map = {
    '&': '&amp;',
    '<': '&lt;',
    '>': '&gt;',
    '"': '&quot;',
    "'": '&#039;'
  };
  return str.replace(/[&<>"']/g, m => map[m]);
}
```

#### （2）CSRF（跨站请求伪造）
**原理：** 诱导用户在已登录状态下访问恶意网站，执行非预期操作

**防御措施：**
- 使用 CSRF Token
- 验证 Referer/Origin 头
- 使用 SameSite Cookie 属性
- 关键操作要求二次验证

#### （3）点击劫持（Clickjacking）
**原理：** 使用透明 iframe 覆盖在正常页面上，诱导用户点击

**防御措施：**
- 设置 X-Frame-Options 响应头
- 使用 CSP 的 frame-ancestors 指令
- JavaScript 检测页面是否被嵌入

#### （4）SQL 注入（通过前端接口）
**原理：** 通过构造特殊输入参数，攻击后端数据库

**防御措施：**
- 前端验证输入格式
- 后端使用参数化查询
- 最小权限原则

#### （5）文件上传漏洞
**原理：** 上传恶意文件（木马、病毒等）

**防御措施：**
- 限制文件类型和大小
- 文件重命名
- 存储到非 Web 目录
- 扫描文件内容

### 6. 前端网站常规优化方案有哪些？

## ECharts 图表库

### 7. Echarts 市面上常见的难点有哪些？

ECharts 作为主流的数据可视化库，在实际项目中常遇到以下难点：

#### （1）大数据量渲染性能问题
**问题描述：** 当数据点超过 10,000 个时，页面卡顿严重

**解决方案：**
```javascript
// ✅ 方案一：开启 dataZoom 和采样
option = {

    { type: 'slider' }
  ],
  series: [{
    type: 'line',
    sampling: 'lttb', // 降采样算法
    data: largeDataArray

// ✅ 方案二：使用 progressive 渐进渲染
option = {
  series: [{
    type: 'scatter',
    progressive: 1000, // 每帧渲染 1000 个点
    progressiveThreshold: 5000 // 超过 5000 个点启用渐进渲染
  }]
};

// ✅ 方案三：使用 canvas 替代 svg（默认就是 canvas）
// ECharts 默认使用 Canvas，性能优于 SVG
```

#### （2）动态数据更新闪烁问题
**问题描述：** 实时数据更新时图表频繁重绘，出现闪烁

**解决方案：**
```javascript
// ❌ 错误做法：每次更新都调用 setOption 完全替换
chart.setOption(newOption);

// ✅ 正确做法：使用 notMerge 和 replaceMerge
chart.setOption({
  series: [{
    data: newData
  }]

  notMerge: false, // 合并配置，避免完全重绘
  replaceMerge: ['series'] // 仅替换系列数据
});

// ✅ 更优方案：使用 appendData 追加数据
chart.appendData({
  seriesIndex: 0,
  data: newDataChunk
```

#### （4）自定义 tooltip 样式和内容

## ES6+ 新特性

### 8. ES6 新特性都有哪些？

ES6（ECMAScript 2015）是 JavaScript 历史上最重要的更新之一，引入了大量新特性。

#### （1）变量声明：let 和 const
```javascript
// var 的问题：函数作用域、变量提升
console.log(a); // undefined（不会报错）
var a = 10;

// let：块级作用域，不存在变量提升
if (true) {
  let b = 20;
}
console.log(b); // ReferenceError: b is not defined

// const：常量声明，必须初始化
const PI = 3.14159;
PI = 3; // TypeError: Assignment to constant variable

// const 声明的对象可以修改属性
const obj = { name: 'Alice' };
obj.name = 'Bob'; // ✅ 允许
obj = {}; // ❌ 不允许重新赋值
```

#### （2）箭头函数（Arrow Functions）
```javascript
// 传统函数
const add1 = function(a, b) {
  return a + b;
};

// 箭头函数
const add2 = (a, b) => a + b;

// 关键特性：this 继承自外层作用域
function Timer() {
  this.seconds = 0;
  
  // ❌ 传统函数 this 指向 window
  setInterval(function() {
    this.seconds++; // this 不是 Timer 实例
  }, 1000);
  
  // ✅ 箭头函数 this 指向 Timer 实例
  setInterval(() => {
    this.seconds++; // this 正确指向 Timer
  }, 1000);
}

// 注意：箭头函数没有 arguments、prototype，不能作为构造函数
```

#### （14）Proxy 和 Reflect
```javascript

  get(target, prop) {
    console.log(`Getting ${prop}`);
    return prop in target ? target[prop] : 'default';
  },
  set(target, prop, value) {
    console.log(`Setting ${prop} to ${value}`);
    target[prop] = value;
    return true;
  }
};

const proxy = new Proxy({}, handler);

console.log.proxy.age); // Getting age \n default

// Reflect - 提供默认行为
```
