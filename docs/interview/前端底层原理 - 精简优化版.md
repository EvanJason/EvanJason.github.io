# 前端底层原理（精简优化版）

参考：https://github.com/KieSun/all-of-frontend

## 一、浏览器与渲染

### 1. 浏览器渲染流程

**完整流程：**
```
HTML → DOM Tree
CSS → CSSOM Tree
     ↓
  Render Tree → Layout → Paint → Display
```

**示例代码：**
```html
<!DOCTYPE html>
<html>
<head>
  <style>
    .container { width: 800px; }
    h1 { color: blue; }
  </style>
</head>
<body>
  <div class="container">
    <h1>Hello World</h1>
  </div>
</body>
</html>
```

**解析过程：**
1. HTML 解析器生成 DOM Tree
2. CSS 解析器生成 CSSOM Tree
3. 合并为 Render Tree
4. 布局（计算位置和大小）
5. 绘制（填充像素）

### 2. 回流（Reflow）vs 重绘（Repaint）

#### Reflow（回流）- 布局改变
**触发场景：**
- 改变窗口大小
- 改变文字大小或内容
- 激活伪类（如 `:hover`）
- 操作 class 属性
- 脚本操作 DOM
- 计算 `offsetWidth` / `offsetHeight`
- 设置 style 属性

**直观示例：**
```javascript
// ❌ 会导致多次回流的写法
const box = document.getElementById('box');
box.style.width = '100px';    // 回流 1
box.style.height = '200px';   // 回流 2
box.style.margin = '10px';    // 回流 3

// ✅ 优化方案 1：批量修改（cssText）
const box = document.getElementById('box');
box.style.cssText = 'width: 100px; height: 200px; margin: 10px;';
// 只触发 1 次回流

// ✅ 优化方案 2：DocumentFragment
const fragment = document.createDocumentFragment();
for (let i = 0; i < 100; i++) {
  const div = document.createElement('div');
  fragment.appendChild(div);
}
document.body.appendChild(fragment);
// 只触发 1 次回流（100 个元素一次性插入）

// ✅ 优化方案 3：使用 transform 替代定位
.box {
  /* ❌ 会触发回流 */
  top: 100px;
  left: 200px;
  
  /* ✅ 不会触发回流（合成层动画） */
  transform: translate(200px, 100px);
}
```

#### Repaint（重绘）- 样式改变
不影响布局的样式变化（背景色、文字颜色），速度较快。

```javascript
// 只会触发重绘的操作
box.style.backgroundColor = 'red';  // 重绘
box.style.color = 'blue';           // 重绘
box.style.visibility = 'hidden';    // 重绘
```

**性能对比：** 回流成本 >> 重绘成本

### 3. BFC（块级格式化上下文）

**定义：** 独立的渲染区域，内部元素不会影响外部布局。

**触发条件（满足任一即可）：**
- 根元素（html）
- float 不为 none
- position 为 absolute/fixed
- display 为 inline-block/table-cell/flex/grid
- overflow 不为 visible（hidden/auto/scroll）

**应用场景 1：清除浮动**
```html
<!-- ❌ 问题：父元素高度塌陷 -->
<div class="parent">
  <div class="child" style="float: left;">浮动元素</div>
</div>

<!-- ✅ 解决方案：父元素触发 BFC -->
<style>
.parent {
  overflow: hidden; /* 触发 BFC */
}
.child {
  float: left;
}
</style>
```

**应用场景 2：两栏自适应布局**
```html
<style>
.left {
  float: left;
  width: 200px;
}
.right {
  overflow: hidden; /* 触发 BFC，不与 left 重叠 */
}
</style>
<div class="left">左侧固定</div>
<div class="right">右侧自适应</div>
```

**应用场景 3：解决 margin 重叠**
```css
/* ❌ margin 重叠 */
.box1 { margin-bottom: 20px; }
.box2 { margin-top: 20px; }
/* 实际间距是 20px，不是 40px */

/* ✅ 让其中一个触发 BFC */
.box2 {
  overflow: hidden;
  margin-top: 20px;
}
/* 现在有真正的 40px 间距 */
```

### 5. 原型链和原型

```javascript
// 原型：任何对象都有原型对象，由 _proto_ 指向构造函数的 prototype
// 原型链：访问变量时找不到会一直访问 _proto_，形成原型链
```

### 6. new 操作符的过程

1. 创建一个新对象
2. 链接到原型（`__proto__` 指向 prototype）
3. 绑定 this（指向新对象）
4. 返回新对象

**手写实现：**
```javascript
function myNew(fn, ...args) {
  // 1. 创建一个新对象，继承构造函数的 prototype
  const obj = Object.create(fn.prototype);
  
  // 2. 绑定 this 并执行构造函数
  const result = fn.apply(obj, args);
  
  // 3. 返回对象（如果构造函数返回的是对象）
  return result instanceof Object ? result : obj;
}

// 使用示例
function Person(name) {
  this.name = name;
}
const p = myNew(Person, 'John');
console.log(p.name); // John
```

### 7. 闭包

**定义：** 内层函数访问外层函数的作用域。

**直观示例 1：计数器**
```javascript
function createCounter() {
  let count = 0; // 私有变量，被闭包保存
  
  return function() {
    count++;
    console.log(count);
  };
}

const counter = createCounter();
counter(); // 1
counter(); // 2
counter(); // 3
```

**直观示例 2：数据缓存**
```javascript
function createCache() {
  const cache = {};
  
  return {
    get(key) { return cache[key]; },
    set(key, value) { cache[key] = value; }
  };
}

const myCache = createCache();
myCache.set('name', 'John');
console.log(myCache.get('name')); // John
```

**直观示例 3：防抖函数**
```javascript
function debounce(fn, delay = 300) {
  let timer = null; // 被闭包保存
  
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 使用场景：搜索框
searchInput.addEventListener('input', debounce(function(e) {
  console.log('发送搜索请求:', e.target.value);
}, 300));
```

### 8. 变量提升和函数提升

**优先级：** 函数提升 > 变量提升

```javascript
// 示例 1
console.log(foo); // function foo() {}
foo = 3;
console.log(foo); // 3
function foo() {}

// 示例 2（let/const 暂时性死区）
console.log(test); // ReferenceError
let test = 1;

const test2 = 2;
test2 = 3; // TypeError: 常量不可重新赋值
```

### 9. call、apply、bind

**共同点：** 都用于改变函数的 this 指向

**区别：**
- `call(fn, arg1, arg2)` - 参数列表
- `apply(fn, [args])` - 参数数组
- `bind(fn)` - 返回新函数，稍后调用

**示例：**
```javascript
var obj = { x: 81 };
var foo = {
  getX: function() { return this.x; }
};

foo.getX.bind(obj)();  // 81
foo.getX.call(obj);    // 81
foo.getX.apply(obj);   // 81
```

## 三、异步编程

### 10. Promise

**三种状态：** pending、resolved、rejected

**特点：**
- 状态不受外界影响
- 状态只能从 pending → resolved/rejected（不可逆）
- then/catch 返回 Promise，支持链式调用

**静态方法：**
```javascript
// Promise.all - 所有成功才成功
Promise.all([p1, p2, p3]).then(results => {
  // results = [result1, result2, result3]
});

// Promise.allSettled - 等待所有完成
Promise.allSettled([p1, p2]).then(results => {
  // [{status: 'fulfilled', value: ...}, {status: 'rejected', reason: ...}]
});

// Promise.any - 一个成功就成功
Promise.any([p1, p2]).then(result => {
  // 第一个成功的结果
});

// Promise.race - 竞速
Promise.race([p1, p2]).then(result => {
  // 第一个完成的结果（不管成功失败）
});
```

**链式调用示例：**
```javascript
new Promise((resolve) => {
  setTimeout(() => resolve(1), 500);
})
.then((res) => {
  console.log(res); // 1
  return new Promise((resolve) => {
    setTimeout(() => resolve(2), 500);
  });
})
.then((res) => {
  console.log(res); // 2
});
```

### 11. async/await

**本质：** Generator + Promise 的语法糖

```javascript
const getData = () => new Promise((resolve) => 
  setTimeout(() => resolve('data'), 1000)
);

async function test() {
  const data = await getData();
  console.log('data: ', data);
  const data2 = await getData();
  console.log('data2: ', data2);
  return 'success';
}

test().then((res) => console.log(res));
// 1 秒后打印 data，再过 1 秒打印 data2，最后打印 success
```

### 12. 事件循环与宏任务/微任务

**执行顺序：** 一个宏任务 → 所有微任务 → 下一个宏任务 → 循环

**宏任务：** script、setTimeout、setInterval、I/O、UI 渲染

**微任务：** Promise.then、MutationObserver、process.nextTick

**经典面试题：**
```javascript
console.log('1. start');

setTimeout(() => {
  console.log('2. timeout');
}, 0);

Promise.resolve()
  .then(() => {
    console.log('3. promise1');
    return Promise.resolve();
  })
  .then(() => {
    console.log('4. promise2');
  });

console.log('5. end');

// 输出顺序：
// 1. start
// 5. end
// 3. promise1
// 4. promise2
// 2. timeout
```

**执行过程分析：**
1. 执行 script，打印 "start"
2. 遇到 setTimeout，回调加入宏任务队列
3. 遇到 Promise，then 回调加入微任务队列
4. 继续执行 script，打印 "end"
5. script 完毕，执行微任务队列
6. 执行第一个 then，打印 "promise1"，返回新 Promise
7. 新 Promise 的 then 也加入微任务队列，立即执行，打印 "promise2"
8. 微任务清空，从宏任务队列取出 setTimeout，打印 "timeout"

## 四、框架原理

### 13. Vue 生命周期

**Vue 2 生命周期：**
```javascript
beforeCreate     // 实例初始化，data/methods 不可用
created          // 实例创建完成，data/methods 可用，DOM 未生成
beforeMount      // 模板编译/挂载之前
mounted          // DOM 已生成，可操作 DOM
beforeUpdate     // 数据更新，虚拟 DOM 打补丁之前
updated          // 虚拟 DOM 重新渲染之后
beforeDestroy    // 实例销毁之前，清定时器/解绑事件
destroyed        // 实例已销毁
```

**Vue 3 生命周期变化：**
- `beforeCreate` + `created` → `setup()`
- `beforeDestroy` → `onBeforeUnmount`
- `destroyed` → `onUnmounted`

### 14. React 生命周期（类组件）

```javascript
// 挂载阶段
componentWillMount      // ⚠️ 已废弃
componentDidMount       // 第一次渲染后（发请求/操作 DOM）

// 更新阶段
componentWillReceiveProps // ⚠️ 已废弃
shouldComponentUpdate     // 控制是否更新
componentWillUpdate       // ⚠️ 已废弃
componentDidUpdate        // 更新后调用

// 卸载阶段
componentWillUnmount      // 移除前（清理工作）

// 新增（React 16+）
static getDerivedStateFromProps(props, state)
getSnapshotBeforeUpdate(prevProps, prevState)
```

### 15. Vue 双向数据绑定原理

#### Vue 2：Object.defineProperty

**原理：** 通过 getter/setter 劫持数据访问和修改。

**简化实现：**
```javascript
class Observer {
  constructor(data) {
    this.walk(data);
  }
  
  defineReactive(obj, key, val) {
    const dep = new Dep();
    
    Object.defineProperty(obj, key, {
      get() {
        Dep.target && dep.addSub(Dep.target); // 收集依赖
        return val;
      },
      set(newVal) {
        if (newVal === val) return;
        val = newVal;
        dep.notify(); // 通知更新
      }
    });
  }
}

// 局限：
// 1. 无法检测对象属性的添加/删除
// 2. 无法监控数组下标变化
// 需要用 Vue.set() / this.$set()
```

**示例：**
```javascript
// ❌ 无法检测
const obj = { a: 1 };
obj.b = 2; // 不会触发视图更新

// ✅ 解决方案
Vue.set(obj, 'b', 2);

// ❌ 数组下标
const arr = [1, 2, 3];
arr[0] = 100; // 不会触发更新

// ✅ 数组方法
arr.push(4); // 可以触发（Vue 重写了数组方法）
```

#### Vue 3：Proxy

**优势：** 直接代理整个对象，支持属性增删和数组下标。

**简化实现：**
```javascript
function reactive(target) {
  return new Proxy(target, {
    get(target, key, receiver) {
      track(target, key); // 收集依赖
      return Reflect.get(target, key, receiver);
    },
    set(target, key, value, receiver) {
      trigger(target, key); // 触发更新
      return Reflect.set(target, key, value, receiver);
    }
  });
}

// 使用示例
const state = reactive({ count: 0 });

effect(() => {
  console.log(state.count); // 自动追踪依赖
});

state.count++; // 自动触发 effect
```

### 16. Virtual DOM 与 Diff 算法

**Virtual DOM：** 用 JavaScript 对象描述 DOM 结构。

**示例：**
```javascript
// 真实 DOM
<div id="app" class="container">
  <h1>Hello</h1>
</div>

// Virtual DOM
{
  type: 'div',
  props: { id: 'app', className: 'container' },
  children: [{
    type: 'h1',
    props: {},
    children: ['Hello']
  }]
}
```

**Diff 策略：**
1. 同层比较（不跨层级）
2. 相同 type 继续比较，不同 type 直接替换
3. key 相同时复用节点

**React Fiber 改进：**
- **React 15**：递归 Diff（同步、不可中断）
- **React 16+**：Fiber 链表遍历（异步、可中断、分优先级）

## 五、框架对比与通信

### 17. Vue vs React 的区别

| 维度 | Vue | React |
|------|-----|-------|
| **数据可变性** | 响应式（数据可变） | 函数式（单向数据流，setState） |
| **开发方式** | 模板语法（HTML/CSS/JS 分离） | JSX（All in JS） |
| **组件写法** | 声明式（Options API） | 类式/函数式（Class/Function） |
| **状态管理** | Vuex（内置） | Redux（社区） |
| **学习曲线** | 较低（易上手） | 较高（灵活性强） |

### 18. Vue 组件通信方式

1. **props / $emit** - 父子组件
2. **$attrs / $listeners** - 跨级组件（Vue 2）
3. **provide / inject** - 祖先与后代
4. **Vuex / Pinia** - 全局状态
5. **$parent / $children** - 访问实例
6. **Event Bus** - 任意组件（Vue 2 推荐，Vue 3 已移除）
7. **ref** - 父访问子

### 19. computed vs watch

| 特性 | computed | watch |
|------|----------|-------|
| **缓存** | ✅ 支持 | ❌ 不支持 |
| **异步** | ❌ 不支持 | ✅ 支持 |
| **返回值** | ✅ 必须有 | ❌ 不需要 |
| **场景** | 多对一（多个数据影响一个） | 一对多（一个数据影响多个） |

**示例：**
```javascript
// computed - 多对一
computed: {
  fullName() {
    return this.firstName + ' ' + this.lastName;
  }
}

// watch - 一对多
watch: {
  searchText(newVal) {
    this.fetchResults(newVal);
    this.updateHistory(newVal);
  }
}
```

### 20. v-if vs v-show

| 特性 | v-if | v-show |
|------|------|--------|
| **DOM** | 条件假时不在 DOM 中 | 始终在 DOM（display:none） |
| **开销** | 切换开销大 | 初始渲染开销大，切换小 |
| **场景** | 条件很少改变 | 频繁切换 |

## 六、网络与安全

### 21. HTTP 状态码

**常见状态码：**

**2xx 成功**
- 200 OK - 成功
- 201 Created - 已创建资源
- 206 Partial Content - 部分内容（断点续传）

**3xx 重定向**
- 301 Moved Permanently - 永久移动（会缓存）
- 302 Found - 临时移动（不缓存）
- 304 Not Modified - 协商缓存

**4xx 客户端错误**
- 400 Bad Request - 参数有误
- 401 Unauthorized - 未授权
- 403 Forbidden - 禁止访问
- 404 Not Found - 未找到

**5xx 服务器错误**
- 500 Internal Server Error
- 502 Bad Gateway
- 503 Service Unavailable
- 504 Gateway Timeout

### 22. 三次握手与四次挥手

**三次握手（建立连接）：**
1. 客户端 → 服务器：SYN=j
2. 服务器 → 客户端：SYN=k + ACK=j+1
3. 客户端 → 服务器：ACK=k+1

**四次挥手（断开连接）：**
1. 客户端 → 服务器：FIN=u（我要关闭）
2. 服务器 → 客户端：ACK=u+1（知道了）
3. 服务器 → 客户端：FIN=w（我也要关闭）
4. 客户端 → 服务器：ACK=w+1（好的）

### 23. 前端安全攻击

#### XSS（跨站脚本攻击）
**防御：** HTML 实体编码、CSP、HttpOnly Cookie

```javascript
function escapeHtml(str) {
  const map = {
    '&': '&amp;',
    '<': '&lt;',
    '>': '&gt;',
    '"': '&quot;',
    "'": '&#039;'
  };
  return str.replace(/[&<>"']/g, m => map[m]);
}

// 使用示例
const userInput = '<script>alert("xss")</script>';
const safeOutput = escapeHtml(userInput);
// &lt;script&gt;alert(&quot;xss&quot;)&lt;/script&gt;
```

#### CSRF（跨站请求伪造）
**防御：** CSRF Token、验证 Referer、SameSite Cookie

#### 点击劫持
**防御：** X-Frame-Options、CSP frame-ancestors

### 25. 时间复杂度

**常见复杂度对比：** O(1) < O(logn) < O(n) < O(nlogn) < O(n²)

**示例：**
- O(1)：常数阶（访问数组元素）
- O(logn)：二分查找
- O(n)：线性查找
- O(nlogn)：归并排序、快速排序
- O(n²)：冒泡排序（双重循环）

### 30. 防抖与节流

**防抖（debounce）：** n 秒内只执行最后一次

```javascript
function debounce(fn, delay = 300) {
  let timer = null;
  
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 使用场景：搜索框输入
searchInput.addEventListener('input', debounce(function(e) {
  console.log('发送搜索请求:', e.target.value);
}, 300));
```

**节流（throttle）：** n 秒内只执行一次

```javascript
function throttle(fn, interval = 300) {
  let lastTime = 0;
  
  return function(...args) {
    const now = Date.now();
    if (now - lastTime >= interval) {
      lastTime = now;
      fn.apply(this, args);
    }
  };
}

// 使用场景：滚动加载更多
window.addEventListener('scroll', throttle(function() {
  console.log('检查是否到底部');
}, 200));
```

### 32. Hash 路由 vs History 路由

| 特性 | Hash | History |
|------|------|---------|
| **URL** | 带 # | 不带 # |
| **兼容性** | 好（支持老浏览器） | 差（需 HTML5） |
| **后端要求** | 无需配置 | 需配置（否则 404） |
| **API** | hashchange | pushState/replaceState |

**示例：**
```javascript
// Hash 路由
window.location.hash = '#/home';
window.addEventListener('hashchange', () => {
  console.log(window.location.hash);
});

// History 路由
history.pushState({}, '', '/home');
window.addEventListener('popstate', () => {
  console.log(location.pathname);
});
```

### 实现 Promise（简化版）

```javascript
class MyPromise {
  constructor(executor) {
    this.status = 'pending';
    this.value = undefined;
    this.reason = undefined;
    this.onResolvedCallbacks = [];
    this.onRejectedCallbacks = [];
    
    const resolve = (value) => {
      if (this.status === 'pending') {
        this.status = 'resolved';
        this.value = value;
        this.onResolvedCallbacks.forEach(fn => fn());
      }
    };
    
    const reject = (reason) => {
      if (this.status === 'pending') {
        this.status = 'rejected';
        this.reason = reason;
        this.onRejectedCallbacks.forEach(fn => fn());
      }
    };
    
    try {
      executor(resolve, reject);
    } catch (e) {
      reject(e);
    }
  }
  
  then(onFulfilled, onRejected) {
    // 简化版实现
  }
}
```
